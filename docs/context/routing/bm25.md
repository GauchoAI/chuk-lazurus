# Routing Strategy: bm25

**CLI flag:** `--strategy bm25`
**Index required:** None (decodes window tokens on the fly)
**Speed:** ~10ms
**Best for:** Exact keyword matching; entity names; known terms

## How it works

Standard BM25 scoring over decoded window text. For each window, decodes the stored token IDs back to text, computes term frequency, and scores against the query using BM25 with IDF weighting.

```
score(w, q) = Σ_t IDF(t) × (tf(t,w) × (k1+1)) / (tf(t,w) + k1 × (1 - b + b × |w|/avgdl))
```

Parameters: k1=1.5, b=0.75 (standard BM25 defaults).

## Strengths

- Fast (~10ms, no forward pass needed)
- Exact match — finds windows containing specific entity names or terms
- No model inference required at routing time
- Works without any extraction phases (only needs windows + tokens)

## Limitations

- No semantic understanding — "lunar module" won't match "Eagle" (the LM's name for it)
- Sensitive to tokenization artifacts in decoded text
- Decodes all windows at query time (cached after first call)

## Usage

```bash
lazarus context generate \
    --model google/gemma-3-4b-it \
    --checkpoint ./ctx/ \
    --prompt "What about Armstrong?" \
    --strategy bm25
```
